# 권한(Role) 정책 가이드

본 프로젝트에서는 **3가지 기본 권한(Role)**을 사용한다.

```
USER
SELLER
ADMIN
```

권한은 `role` 테이블과 `user_role` 테이블을 통해 관리된다.

---

# 1. USER

일반 **구매자 계정**이다.

가능 기능

```
상품 조회
상품 검색
장바구니 사용
주문 생성
결제 진행
리뷰 작성
쿠폰 사용
포인트 사용
```

설명

```
쇼핑몰을 사용하는 일반 사용자
```

---

# 2. SELLER

상품을 판매하는 **판매자 계정**이다.

가능 기능

```
상품 등록
상품 수정
상품 삭제
옵션 관리
재고 관리
주문 확인
배송 처리
정산 조회
```

설명

```
쇼핑몰에 입점하여 상품을 판매하는 사용자
```

주의

```
SELLER 권한은 일반 USER 계정에 추가로 부여될 수 있다.
```

---

# 3. ADMIN

플랫폼을 운영하는 **관리자 계정**이다.

가능 기능

```
회원 관리
판매자 승인
상품 검수
주문 관리
환불 관리
정산 관리
쿠폰 관리
시스템 운영
```

설명

```
쇼핑몰 전체 시스템을 관리하는 운영 계정
```

---

# 4. 권한 구조

권한은 다음 구조로 관리된다.

```
user
 └ user_role
     └ role
```

설명

```
한 명의 사용자는 여러 개의 권한을 가질 수 있다.
```

예시

```
USER
USER + SELLER
ADMIN
```

---

# 5. 권한 관리 정책

기본 규칙

```
회원 가입 시 기본 권한은 USER이다.
```

판매자 권한

```
USER → SELLER 권한 추가
```

관리자 권한

```
ADMIN은 시스템에서 직접 부여한다.
```

---

# 6. Role 테이블 데이터 예시

```
id | role_name
1  | USER
2  | SELLER
3  | ADMIN
```

---

# 7. UserRole 테이블 예시

```
user_id | role_id
1       | 1
1       | 2
```

설명

```
user_id 1 은 USER + SELLER 권한을 가진다.
```

---

# 정리

```
USER   → 구매자
SELLER → 판매자
ADMIN  → 관리자
```

권한은 `role`, `user_role` 테이블을 통해 관리되며
사용자는 **여러 권한을 동시에 가질 수 있다.**
